The Five Domains

What each section measures

Every domain is mapped directly to a cognitive skill that CS curricula demand. No filler, no general knowledge — every question earns its place.

01

Abstract Logical Reasoning

Deductive logic, propositional inference, syllogisms, and sequence completion. Tests whether you can navigate formal logical systems — the foundation of algorithm design, compiler theory, and formal verification.

15 Questions Logic · Patterns · Sequences
02

Mathematical Reasoning

Discrete mathematics, set theory, propositional logic, basic induction, and numerical reasoning. CS is fundamentally applied mathematics — this measures how fluently you can operate in the mathematical substrate of computation.

15 Questions Sets · Logic · Numbers
03

Computational Thinking

Flowchart tracing, loop invariants, pseudocode debugging, and problem decomposition. Can you think like a machine — without one in front of you? This is the language of algorithms before syntax enters the picture.

15 Questions Algorithms · Flowcharts · Loops
04

Spatial / Temporal Reasoning

Mental rotation of tree structures, memory layout visualisation, and reasoning about recursive and sequential processes in time. Foundational for understanding data structures, operating systems, and network protocols.

15 Questions Recursion · Structures · Sequences
05

Attention to Precision

Spot-the-difference in nested conditions, exact rule following, and precision in edge-case reasoning. CS has no tolerance for near-correct. This domain tests whether you are wired for an environment where close is the same as wrong.

15 Questions Precision · Edge Cases · Rules
